The Idea That Started Everything
If you've spent any time on Polymarket, you know there are two kinds of markets:
The spicy ones like elections, sports, drama (they have high volatility and wild swings). Professional traders are waiting here to take your money. You do not want to be in there without a serious edge.
Then we have the boring ones with markets like:
"Will the US 10-year Treasury yield be above 3% at end of month?"
"Will the Fed hold rates at the May 2026 meeting?"
"Will the US avoid a recession in Q3 2025?"
These are bond and macro markets that are basically already resolved by the time you're reading them. The Fed already met, and the yield already printed. Everyone literally knows the answer (the market just hasn't officially expired yet).
And in that gap between "everyone knows" and "contract settles," something actually interesting happens; the YES price sits at 98 or 99 cents, not 100 yet, because settlement hasn't happened. But the outcome is certain.
Best kind of money made is slow and boring
That 1–2 cent spread is free money. Slow, boring, almost-guaranteed money. The problem is finding these markets fast enough before they fill or the price moves to resolution.
Doing it manually is tedious and error-prone, so I decided to automate it.

What Claude Built
The Scanner:
The scanner hits Polymarket's Gamma API and pages through every active market. It uses cursor-based pagination i.e the current V2 standard, where each response returns a next_cursor token you pass into the next request, looping until no cursor is returned. This replaced the old offset system that broke with the CLOB V2 upgrade.
python
# Cursor-based pagination — Polymarket CLOB V2
async def _fetch_all_markets(self) -> list[dict]:
    all_markets = []
    next_cursor = None

    while True:
        params = {
            "active":   "true",
            "closed":   "false",
            "limit":    GAMMA_PAGE_LIMIT,
        }
        if next_cursor:
            params["next_cursor"] = next_cursor

        resp = await self._client.get(
            f"{POLYMARKET_GAMMA_URL}/markets/keyset",
            params=params,
        )
        resp.raise_for_status()
        data = resp.json()

        markets = data.get("markets", [])
        all_markets.extend(markets)

        next_cursor = data.get("next_cursor")
        if not next_cursor:
            break

        await asyncio.sleep(GAMMA_RATE_LIMIT_SLEEP)

    return all_markets

First, a hard numeric filter, the market must have a YES price of at least 97 cents, volume above $10,000 (to confirm the price is real and not a liquidity glitch), and between a few hours and 14 days until expiry. The Analyzer
Every candidate gets scored on a 0–100 scale. Claude designed a five-component formula with weights that reflect how much each factor actually signals certainty:
Price dominates because that's the strongest signal of certainty.
 A market at 0.998 is more certain than one at 0.970.
Volume is the second most important factor because a low-volume 0.99 price can be a mirage, just one person's stale limit order sitting on the book.
Liquidity confirms you can actually enter at the shown price.
Time-to-expiry shrinks the window for surprise outcomes.
Markets scoring 70+ get alerted.
Markets at 85+ get the HIGH CONFIDENCE label

This is the detail I wouldn't have thought of on my own. Claude pointed out that if the bot runs every 30 minutes and a market stays at 0.98 for two days before settling, I'd get 96 Telegram alerts about the same market. The cache doubles as a full alert history. Every alert ever sent is in the database with the confidence score, price, volume, and Telegram message ID all feeding back into the dashboard.

The Rust Scanner
After the Python bot was working, I asked Claude: "Is there a faster way to do the API scanning?" It explained that even with async HTTP, Python still fetches pages sequentially, one cursor hop at a time. It offered to write a Rust companion that handles the cursor loop natively with tokio, making it roughly 10x faster end-to-end.
rust
// Rust cursor-based scanner — CLOB V2
async fn fetch_all_markets(client: &Client, args: &Args) -> Result<Vec<GammaMarket>> {
    let mut all_markets: Vec<GammaMarket> = Vec::new();
    let mut next_cursor: Option<String> = None;

    loop {
        let mut params = vec![
            ("active",  "true"),
            ("closed",  "false"),
            ("limit",   "100"),
        ];
        let cursor_val;
        if let Some(ref c) = next_cursor {
            cursor_val = c.clone();
            params.push(("next_cursor", &cursor_val));
        }

        let resp = client
            .get(&format!("{}/markets/keyset", GAMMA_BASE_URL))
            .query(¶ms).send().await?
            .json::<KeysetResponse>().await?;

        all_markets.extend(resp.markets);
        next_cursor = resp.next_cursor;
        if next_cursor.is_none() { break; }

        tokio::time::sleep(Duration::from_millis(200)).await;
    }
    Ok(all_markets)
}
The Rust binary outputs JSON to stdout; Python reads it via subprocess. The two languages never directly touch each other's code. The Rust scanner is a drop-in replacement, switched with a single flag:
bash
python main.py --rust   # use Rust scanner (~10x faster)
python main.py          # fallback to Python scanner
I didn't write a single line of Rust myself. I described what I wanted: a fast scanner that outputs the same JSON format as the Python one, and Claude wrote it. Compiles with cargo build --release to a single binary under 5MB.

What I've Learned Running It
The obvious markets fill fast: If it's a screaming-obvious outcome Fed decision day, major treasury auction, the YES price is often already at 0.995 by the time my 30-minute scan catches it. The real edge is markets that should be obvious but haven't been bid up yet, usually because they're lower volume or more obscure.
Volume matters more than price alone: A 0.98 market with $500k Volume is a better opportunity than a 0.99 market with $8k volume. Low volume means the price might be stale. The confidence formula accounts for this, with a high-volume 0.98 score comparable to a low-volume 0.99.
The sweet spot is 1–5 days to expiry: Close enough that the outcome is clearly priced, far enough that there's still spread to capture. Markets expiring in hours have essentially zero spread left. Markets expiring in two weeks still carry real uncertainty risk, even at 0.97.
It's not about getting rich fast: A 2% return on $500 is $10. But rotating through 4–5 of these per week adds up without the volatility of trying to predict sports outcomes or political events.
