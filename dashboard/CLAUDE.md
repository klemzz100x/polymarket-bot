# Dashboard

## Mission

Construire une interface opérateur temps réel pour superviser :
- trading,
- risque,
- wallets,
- alpha,
- monitoring,
- infrastructure.

## Objectifs UX

Le dashboard doit être :
- ultra lisible,
- rapide,
- dense en information utile,
- orienté opérateur quant/HFT.

## Priorités

1. Temps réel
2. Clarté
3. Alerting
4. Monitoring critique
5. Rapidité navigation

## Sections importantes

- PnL
- Risk
- Positions
- Wallet Intelligence
- Liquidity
- Toxic Flow
- Reward Markets
- Execution Health
- Websocket Status
- Error Monitoring

## Contraintes

- Éviter surcharge visuelle.
- Priorité aux informations actionnables.
- Support dark mode.
- Support multi-screen.

## Stack cible

- Next.js
- Tailwind
- shadcn/ui
- Grafana
- WebSockets
